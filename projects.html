Rover

Risk sensitive surveillance using unmanned aerial vehicles

By using simple local rules, we created an algorithm that can provide risk sensitive surveillance over a given area using a team of quadrotors. The algorithm reduces the frequency of surveillance of risky areas and increases its altitude whilst being aware of its sensor quality to avoid the risk.

dCRoPS

Path planning for swarms in dynamic environments

We have developed a novel path planning algorithm for swarms that uses probabilistic roadmaps for global planning and potential fields for local planning. The algorithm is able to plan the movements of a swarm around static and dynamic obstacles in order to reach the goal.

Malt

Multi-agent source localization

Malt is a multi-agent source localization toolkit that is used to determine the location of undirected stimuli by combining the knowledge from each of the agents. The agents move around the area in order reduce the noise in their sensor readings to better predict the location of the source.

Racer

Path planning in dynamic environments by predicting the trajectory of moving obstacles

Racer is a project that I am working on as part of my senior honours project. This algorithm uses the previous movements of dynamic obstacles to build a Markov model of future trajectories that the obstacle will follow. It will use combine this knowledge with multiple planning techniques to create safer, collision free paths for an agent even when the dynamic obstacles are much larger and faster than the agent.

ZeroMQ-ROS

A multi-master middleware for swarm robotics that combines ROS and ZeroMQ

ZeroMQ-ROS is a middleware that is able to control multiple ROS masters from a central node from swarm applications. This project was used by Rover to control the quadrotors in the real-world experiments.

Bowtie

In browser mobile aided sensor acquisition using HTML5

Bowtie is a middleware and protocol that was created that would allow smartphones to become sensors with little effort. By visiting a website and specifing a few configuration details, sensor data would be gathered by HTML5 and sent to a central server, where it can then be retrieved in real time using a RESTful API for robotic applications.
